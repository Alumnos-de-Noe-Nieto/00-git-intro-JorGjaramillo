\## ¿Qué es Git?



Git es un sistema de control de versiones que permite a los programadores tener diferentes versiones de su código, como una "maquina del tiempo" en su código.



\## Por qué necesitamos Git



Sin Git, cuando trabajamos en equipo seria un desastre la organización, se sobrescribiría líneas de código, no se sabría cual es el código correcto y si falla algo, se debería revisar todo el código.



\## Los 3 conceptos clave de Git



1\. \*\*Repositorio:\*\* El espacio donde puedo tener varias versiones de mis códigos o donde puedo colaborar con mi equipo para un proyecto en común.

2\. \*\*Commit:\*\* Es una version actualizada de un archivo o varios, donde se tiene que especificar en un mensaje que se hizo.

3\. \*\*Rama:\*\* Es el espacio donde yo puedo trabajar sin afectar el trabajo principal.



\## Los comandos que aprendí

git add - Guarda uno o varios archivos

git push - Sube los archivos al repositorio

git pull - Descarga los cambios del repositorio

git commit - Guarda los cambios de los archivos y se escribe un mensaje de lo que se hizo

git status - Ver que archivos se modificaron



\## Mi reflexión final



Lo más importante que aprendí de Git es que sin el, hacer proyectos de programación, seria muy difícil el comunicarse con mi equipo, y es una herramienta increíble para tener muchas versiones de mi código propio.
